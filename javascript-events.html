Standard JavaScript Events Including those for Mobile Devices ( Ex. onTouchBegin, onLoad, etc.) and Animation and Transition Events

Differences

When it comes to mobile and non mobile versions of a website there are a ton of similarities. The main difference is that for the most part, mobile devices use touch instead of a mouse so any interaction that involves hover won't register.

Example

When you navigate to the home page there is a loading page that uses jQuery window.load fuction and fades out the page loader.

$(window).load(function () { setTimeout(function(){ $('#preloader').fadeOut('slow', function () { }); },2000); // set the time here });

I set there to be a delay because the page was loading to quickly and didn't give enough time for the user to read.

Here is another example

<style type="text/css"> .box { text-align: center; height: 100px; width: 100px; background-color: lightgrey; } </style> <script type="text/javascript"> window.onload = loaded(); function loaded(){ document.getElementById("test1").innerHTML = "Page has loaded."; } </script> <div class="box" id="loadedDiv" onload="loaded()"></div>
